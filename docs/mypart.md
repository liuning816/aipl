# 题库功能开发计划（我的部分）

## 1. 先定数据契约（字段与接口返回）
状态：已完成（框架版）
先和管理员同学对齐公共数据结构：
- content/status/visibility/stats/tags/report_count 等字段名
- 冻结接口返回格式，避免前端后面反复改

已落地接口：
- GET /api/question-bank/contents
- GET /api/question-bank/contents/:id
- POST /api/question-bank/contents
- POST /api/question-bank/contents/:id/vote
- POST /api/question-bank/contents/:id/report
- PATCH /api/question-bank/contents/:id/visibility

当前返回补充字段：
- my_vote / my_reported / can_edit
- pagination(limit/skip/total/count/has_more)

## 2. 完成题库“浏览与检索”主流程
状态：已完成（可用版）
- 列表页：分页、标签筛选、类型筛选（单题/试卷）
- 详情页：题目/试卷内容完整展示
- 这一步先做“能稳定查看”，不做复杂排序推荐

## 3. 完成“上传”主流程（手动上传 + AI转存入口）
状态：已完成（可用版）
- 单题上传表单跑通
- 试卷上传（题数 > 3）校验
- 私有/公开切换能力跑通
- 从 AI测验 页面加“一键转题库”入口（最小可用版）

已补充：
- Quiz页支持“一键转存为题库试卷”
- Quiz题目卡支持“上传该题到题库”

## 4. 完成互动能力（赞/踩/举报）
状态：已完成（可用版）
- 赞踩互斥、单用户单内容唯一
- 举报理由必填（枚举值）
- 前端状态回显（已赞/已踩/已举报）

## 5. 完成学习路径接入和用户动线
状态：已完成
- 你已经加了“题库测验”按钮，这里补完跳转后的上下文传递（topic/week/subtopic）
- 统一文案和入口：学习资源 -> 题库测验 -> AI测验
- 保证从 roadmap 到题库再返回的路径自然

## 6. 完成基础质量约束（非管理员逻辑）
状态：已完成（第一版）
- 前端表单校验：标签必填、试卷题数校验、标题长度等
- 后端参数校验与错误码统一（400/403/404）
- 防重复提交、空内容提交保护

## 7. 联调与回归测试
状态：进行中
- 用户主链路回归：浏览 -> 上传 -> 点赞/点踩 -> 举报 -> 私有/公开切换
- 与管理员同学联调：被举报内容状态变化后，用户侧展示是否正确（例如 reported/hidden）
- 做一个最小测试清单文档，作为验收依据

最小联调清单：
1. 从学习路径进入题库页，列表可分页加载。
2. 单题上传成功，列表可看到并可切换私有/公开。
3. 试卷上传（>=4题）成功，详情可看到 items。
4. 点赞/点踩互斥，重复点击可撤销。
5. 举报提交后，前端显示“已举报”。
6. Quiz页可一键转存整套试卷。
7. Quiz页可单题一键转存。

## 8. 最后再做“信誉分”挂钩（可并行到后期）
状态：未开始（按计划后置）
- 先只埋点和日志，不急于复杂算法
- 先记录：上传、被赞踩、举报成立与否、内容质量评估输入
- 信誉分规则可以在功能稳定后再启用
