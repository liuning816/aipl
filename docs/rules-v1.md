# AIPL 题库与社区规则文档 v1

## 1. 文档信息
- 版本：v1.0.0
- 日期：2026-03-12
- 适用范围：公共题库、AI测验、题库测验、互动、信誉、治理、管理员
- 定位：`AI测验（个性化） + 题库测验（社区沉淀）` 双引擎学习平台

## 2. 产品目标与边界
- 核心目标：构建高质量公共题库，提升用户贡献积极性，建立可持续治理闭环。
- 闭环定义：`学习 -> 贡献 -> 社区反馈 -> 信誉变化 -> 权益变化 -> 持续贡献`。
- v1 边界：先做规则驱动，不做复杂推荐模型与重竞技系统。

## 3. 名词定义
- 单题：`single_question`
- 试卷：`paper`（题目数量 `> 3`）
- 可见性：`private` / `public`
- 内容状态：`draft` / `published` / `reported` / `hidden` / `deleted`
- 信誉分：内容可信度分值（`0-100`）
- 活跃积分：行为活跃度分值（与信誉分分离）

## 4. 页面与入口规范
- 学习路径页按钮顺序：`学习资源` -> `题库测验` -> `AI测验`
- 原“开始测验”统一更名为“AI测验”。
- 上传入口：
  - 手动上传单题/试卷
  - AI测验结果一键转存到题库

## 5. 内容模型与发布规则
- 内容类型：`single_question`、`paper`
- 可见性：`private`、`public`
- 必填字段：
  - 标题
  - 标签（至少 1 个）
  - 描述
  - 难度
  - 答案与解析
  - 来源（`ai_generated` / `user_original` / `adapted`）
- 发布规则：
  - private 内容仅作者可见
  - public 内容进入公共题库并开放互动
  - public 每日发布额度由信誉等级控制

## 6. 标签规则
- 标签为题库分类主键，上传时必填。
- v1 采用“受控一级标签 + 用户可选二级标签”模式。
- 标签治理由 `tag_manager` 负责新增、合并、下线。

## 7. 互动规则（赞/踩/举报）
- 赞踩规则：
  - 单用户对单内容只允许一个有效状态：`upvote` 或 `downvote` 或 `none`
  - 支持撤销与切换，系统只保留最终状态
  - 禁止重复刷票，命中风控不计入统计
- 举报规则：
  - 单用户可举报同一内容，但只保留最近一次
  - 举报必须选择理由，可附文本说明
  - 理由枚举：`违规内容`、`错误答案`、`抄袭`、`广告或无关`

## 8. 信誉分系统（v1）
- 初始分：`50`
- 分值范围：`0-100`
- 定位：衡量“内容可信度”，不直接奖励纯活跃行为。
- 等级区间：
  - D：`0-39`
  - C：`40-59`
  - B：`60-79`
  - A：`80-100`

## 9. 信誉加减分规则
- 加分项：
  - 被采纳纠错：`+3`
  - 优质解析达标：`+2 ~ +5`
  - 上传内容 7 日质量达标：`+5`
- 扣分项：
  - 上传内容 7 日质量不达标：`-5`
  - 举报成立：`-10`
  - 严重违规：`-30`（可叠加禁发/封禁）
- 防刷约束：
  - 每日可得信誉上限
  - 同类行为阶梯衰减
  - 异常账号或异常群体行为不计分

## 10. 信誉等级权益
- D 级（0-39）：
  - public 内容强审核
  - public 每日发布上限 `1`
- C 级（40-59）：
  - 常规审核
  - public 每日发布上限 `2`
- B 级（60-79）：
  - 先发后抽检
  - public 每日发布上限 `5`
  - 题库排序曝光权重 `+10%`
- A 级（80-100）：
  - 部分免初审（保留举报后复核）
  - public 每日发布上限 `10`
  - 题库排序曝光权重 `+20%`

## 11. 惩罚与恢复机制
- 轻度惩罚：
  - 触发：连续低质量内容
  - 动作：提醒、降权、限流
- 中度惩罚：
  - 触发：连续多条高踩低赞且低完成率
  - 动作：改为先审后发、降低发布额度
- 重度惩罚：
  - 触发：高比例举报成立，或命中严重违规
  - 动作：隐藏/删除、禁发 `3-14` 天、封禁
- 恢复机制：
  - 观察期内持续优质贡献可恢复信誉与额度
  - 支持申诉复核

## 12. 管理员体系与职责
- `content_moderator`：举报处理、上下架、申诉复核
- `tag_manager`：标签规则、标签清理与合并
- `ops_admin`：处罚执行、账号治理、策略调参

## 13. 管理员 SOP（标准流程）
- Step 1：系统风险分拣（高风险优先）
- Step 2：人工复核证据（内容、历史记录、举报理由）
- Step 3：执行动作（通过、隐藏、删除、处罚）
- Step 4：记录审计日志（操作者、对象、动作、理由、时间）
- Step 5：通知用户并开放申诉入口

## 14. 审计与合规
- 审计必记动作：
  - 内容隐藏、恢复、删除
  - 账号警告、禁发、封禁、解封
  - 规则策略变更
- 审计字段：`operatorId`、`targetId`、`action`、`reason`、`createdAt`
- 日志保留期：不少于 `180` 天

## 15. OpenClaw 借鉴方向（v1-v2）
- 原则：借鉴“挑战驱动 + 协作成长”，不照搬重竞技。
- 落地方向：
  - 周主题挑战（按知识点组织）
  - 双榜单：学习榜（完成/正确/持续）与贡献榜（质量/采纳/影响力）
  - 任务链：`完成测验 -> 提交纠错 -> 发布优质内容`
  - 个人影响力展示：内容帮助学习人数
- 约束：所有榜单与奖励优先看质量，不以数量为唯一标准。

## 16. 数据指标（验收口径）
- 供给指标：日新增单题数、日新增试卷数、public 发布率
- 质量指标：举报成立率、纠错采纳率、低质量内容占比
- 参与指标：点赞率、踩率、互动率、上传转化率
- 治理指标：举报处理时长、申诉通过率、误判率
- 留存指标：题库测验 7 日留存、贡献者 7 日留存

## 17. 配置中心（支持动态调参）
- `REPUTATION_INIT = 50`
- `QUALITY_PASS_ADD = 5`
- `QUALITY_FAIL_DEDUCT = 5`
- `REPORT_VALID_DEDUCT = 10`
- `SERIOUS_VIOLATION_DEDUCT = 30`
- `A_LEVEL_THRESHOLD = 80`
- `B_LEVEL_THRESHOLD = 60`
- `C_LEVEL_THRESHOLD = 40`
- `PUBLIC_DAILY_LIMIT_A/B/C/D = 10/5/2/1`

## 18. 实施里程碑
- Phase 1（2 周）：题库 MVP（内容、标签、可见性、赞踩举报）
- Phase 2（2 周）：信誉分 v1、等级权益、AI一键转题库
- Phase 3（2-3 周）：管理员后台 v1、审计日志、申诉流程
- Phase 4（2-3 周）：周挑战、双榜单、影响力卡片
